# 计算机网络 #

#### 储存转发传输 - 传输时间计算 ####

> 传播时延不予考虑

由N条速率均为R的链路组成的路径, 从源到目的地发送一个分组的总体情况, 端到端时延为:

d<sub>端到端</sub> = N·L/R

下面来分析P个分组经过N条链路序列的时延:

令T<sub>(p)</sub>表示第P个分组刚好全部被最后一个路由器转发出去时的时间, 则有如下关系:

>注意:T<sub>(p)</sub>同时表示第P+1个分组刚开始被转发时的时间

T<sub>(p)</sub>=T<sub>(p-1)</sub>+L/R

第一个分组经过N条链路所用时间为:

d<sub>端到端</sub> = N·L/R

即T<sub>(p)</sub>=T<sub>(p-1)</sub>+L/R=T<sub>(p-2)</sub>+2L/R=...=T<sub>(1)</sub>+(p-1)L/R=(N+p-1)L/R

同时,有另一种思考方式可以得出同样的结果:

将传输链路忽略, 只考虑N-1台路由器在其中的作用(原因是忽略了传播时延), 长度为L的分组通过路由器所需时间为2L/R, 若将路由器看作限速为
R的路径, 其长度为L, 则长度为L的绳子以R的速度全部通过这段长度为L路径, 所用时间为2L/R. 

对此进行类比, 若有N条链路, 则其中有N-1台路由器, P个分组的数据总长度为PL, 类比作通过限速为R, 长度为(N-1)L的路径,所需要的时间即为(P+N-1)L/R, 与递推方法得出的公式一致.
